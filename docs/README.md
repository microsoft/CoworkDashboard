# Cowork ROI — Team installer generator (GitHub Pages)

A **static, 100% client-side** page that lets a team's manager/lead bake their Teams channel into the
`cowork-roi-member` skill, so teammates install a ready-to-go copy that posts to the right channel
**without ever being asked for a link** on first run.

Live page (once Pages is enabled): `https://<owner>.github.io/cowork-roi-member/`

## Manager flow

1. Open the page.
2. Paste your team's **Teams channel link** (in Teams: channel **⋯** → **Copy link**).
3. Click **Parse & verify** and confirm the resolved `channel_name` / `team_id` / `channel_id`.
4. Click **Download configured skill (.zip)**. The zip is named
   `cowork-roi-member-<channel-name-or-teamid>.zip`.
5. Share that zip with your team. Each member uploads it in Copilot Cowork and asks it to install the
   skill — it then posts to your channel with no link prompt.

The pasted link is parsed **entirely in the browser**; it is never uploaded. The page makes zero
third-party network calls (JSZip is vendored locally at `docs/jszip.min.js`), and the only network
activity is same-origin fetches of the skill template under `docs/skill-template/`.

## What the zip contains

A single top-level `cowork-roi-member/` folder mirroring the skill exactly — `SKILL.md`, `README.md`,
`scripts/`, `references/`, and `config/` — with `config/team_channel.json` overwritten to hold the
resolved channel. That layout matches how the skill installs from a zip today. If a member is instead
handed the **un-configured** skill (empty `config/team_channel.json`), the skill falls back to the
original ask-on-first-run behavior.

## Files

| Path | Purpose |
| --- | --- |
| `index.html` | One-page form + install instructions. |
| `app.js` | Link resolver + in-browser zip assembly (JSZip). |
| `jszip.min.js` | Vendored JSZip (no CDN). |
| `skill-template/` | Mirror of the skill files the browser fetches, plus `manifest.json`. Generated by the sync workflow — do not hand-edit. |
| `README.md` | This file. |

## One-time GitHub Pages setup (manual)

In the repository: **Settings → Pages → Build and deployment → Source: Deploy from a branch →
Branch: `main` → Folder: `/docs` → Save.**

## How the template stays in sync

`.github/workflows/sync-template.yml` runs on pushes to `main` that change the skill folder. It copies
the current skill into `docs/skill-template/cowork-roi-member/` and regenerates `manifest.json`,
excluding `CHANGELOG-*.md`, `__pycache__`, `*.pyc`, `working/`, and any per-user registry/channel
files. It always writes an **empty** `config/team_channel.json` into the template — a populated
per-team copy is never committed.
