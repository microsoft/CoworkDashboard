The KPI band — what each headline number means

Time saved — expert-equivalent hours Cowork saved the team this period.
Value / cost reduction — those saved hours priced out (hours × hourly rate).
Team speed multiplier — how much faster: expert hours ÷ hands-on hours.
Contributors — how many teammates posted their stats this period.
Sessions — distinct Cowork chats run across the whole team.
Deliverables — files, decks, docs, web pages and other outputs made.
Active days — person-days with at least one Cowork task in the window.
Hands-on time — real time at the keyboard (the assisted clock).

The two controls

Period — pick the reporting window; every number on the page updates to match.
Hourly rate — change $/hr and all value / cost-reduction figures recompute instantly (default $72/hr). Only the pricing changes; hours and counts stay the same.

How task categories are derived

Two different questions are answered by two different fields. Business process = what business need the work served. Task category = how the work was done (the method). Each task is sorted into up to two categories from three signals:

1 · Output file type — spreadsheets (.xlsx / .csv / .json) → Analysis & Research · code & web (.py / .html / .sql / .js) → Write or debug code · documents (.docx / .pdf / .pptx / images) → Document & content creation · packaged skills (.zip / .skill) → Specialized workflows.
2 · Goal keywords — “analyze / research / ROI / benchmark” → Analysis · “debug / refactor / script / API / automation” → Write or debug code · “email / inbox / reply” → Email · “meeting / transcript / standup / agenda” → Meeting.
3 · Input signal — if the sources are data files, or there are 3 or more of them → Analysis.

When more than one matches, they rank (code › analysis › specialized › document › communication › meeting › email › general) and the top two are kept. A task with no saved file and no signal falls to General assistance / Other. Each category then carries a research time band (below); time saved = tasks × band.

Deliverables — and why some show only a file format

Under Work by business process, a deliverable with a de-identified name (e.g. “Team ROI dashboard”) lists on its own row. A row that shows only a format (e.g. “HTML”) is a deliverable whose name wasn't included in that teammate's post — the Member skill can post either a named line or a compact type-only line. It does not mean Cowork failed to recognize the work. To keep the list readable, all type-only deliverables of one format collapse into a single row — e.g. “HTML · 5 deliverables” — with their hours and value summed.

How widespread is a category (reach)

Each task-category row shows how many contributors used it — e.g. “used by 4 of 5 contributors” — so you can see where usage is concentrated vs. spread, not just the volume of hours. This is an aggregate count and never names anyone. To protect a small team, when fewer than 3 people used a category the exact number is withheld and shown as “used by <3 contributors”.

Privacy & anonymity

Nothing is shown at an individual level. Members appear only as counts. The one per-attribute view (category mix) breaks a Role out only when 3+ contributors share it; otherwise they collapse into a single combined bar. The only attribute used is the directory Role (job title) that a teammate's post carries — never names, never country, never file names or prompts.

Value model

Time saved = Σ run tasks × the research band per task category (minutes saved/run). Value = expert-equivalent hours × hourly rate (default $72/hr; adjust live). Speed multiplier = expert ÷ modeled hands-on hours. All figures come from the posts; the dashboard only re-totals and re-prices them.

Value pillars

Revenue Growth (top-line), Cost Reduction (cost base), Risk Mitigation (protect value), Transformation (new ways of working / AI adoption). Shared crosswalk across all three ROI skills.
